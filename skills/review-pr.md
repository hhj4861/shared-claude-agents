# PR Review Skill

Pull Request를 리뷰하는 스킬입니다.

## 사용법

```
/review-pr
/review-pr 123
/review-pr https://github.com/owner/repo/pull/123
```

## 워크플로우

1. **PR 정보 수집**
   - `gh pr view` 또는 URL에서 PR 정보 확인
   - 변경된 파일 목록 확인
   - 커밋 히스토리 확인

2. **코드 분석**
   - 변경사항 diff 분석
   - 코드 품질 체크
   - 보안 취약점 확인
   - 테스트 커버리지 확인

3. **리뷰 작성**
   - 주요 변경사항 요약
   - 개선 제안
   - 질문 사항

## 리뷰 체크리스트

### 코드 품질

```
□ 코드가 읽기 쉬운가?
□ 함수/변수 네이밍이 명확한가?
□ 중복 코드가 있는가?
□ 불필요한 복잡성이 있는가?
```

### 기능성

```
□ 요구사항을 충족하는가?
□ 엣지 케이스가 처리되었는가?
□ 에러 핸들링이 적절한가?
```

### 보안

```
□ SQL Injection 위험이 있는가?
□ XSS 위험이 있는가?
□ 민감 정보 노출이 있는가?
□ 인증/권한 체크가 있는가?
```

### 테스트

```
□ 테스트가 추가되었는가?
□ 기존 테스트가 통과하는가?
□ 테스트 커버리지가 적절한가?
```

### 성능

```
□ N+1 쿼리 문제가 있는가?
□ 불필요한 리렌더링이 있는가?
□ 메모리 누수 가능성이 있는가?
```

## 리뷰 출력 형식

```markdown
## PR 리뷰: #123

### 요약
이 PR은 로그인 기능에 유효성 검사를 추가합니다.

### 좋은 점
- ✅ Zod를 활용한 깔끔한 유효성 검사
- ✅ 에러 메시지가 사용자 친화적

### 개선 제안
- 🔧 `src/components/login-form.tsx:45`
  비밀번호 최소 길이 상수화 권장

- 🔧 `src/lib/validations.ts:12`
  이메일 검증 정규식 재사용 권장

### 질문
- ❓ 로그인 실패 시 로깅이 필요하지 않을까요?

### 결론
**Approve** - 작은 개선 후 머지 가능
```

## 리뷰 결과

| 결과 | 설명 |
|------|------|
| Approve | 머지 가능 |
| Request Changes | 수정 필요 |
| Comment | 의견만 제시 |
